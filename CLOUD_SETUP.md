# 云开发配置指南

## 步骤1：开通云开发

1. 打开微信开发者工具
2. 点击工具栏左侧的 **"云开发"** 按钮
3. 点击 **"开通云开发"**
4. 选择 **"云开发环境"**（不是云托管）
5. 计费方式选择 **"按量付费"**（有免费额度）
6. 点击 **"开通"**（如果有环境名称输入框，可填写如 `genealogy-dev`；如果没有会自动创建）
7. 开通成功后，在云开发控制台顶部找到并复制**环境ID**（格式通常为 `xxx-xxx` 或自动生成的字符串）

## 步骤2：配置小程序

打开 `miniprogram/app.js`，将第7行的环境ID替换为你自己的：

```javascript
// 修改前
env: 'your-env-id',

// 修改后（替换为你的环境ID）
env: 'cloud1-xxx',  // 替换为你的环境ID
```

## 步骤3：创建数据库集合

1. 在开发者工具中，点击 **"云开发"** → **"数据库"**
2. 点击 **"+"** 号创建集合
3. 输入集合名称：`genealogies`
4. 点击 **"确定"**

## 步骤4：设置数据库权限

1. 在 `genealogies` 集合的权限设置中
2. 选择 **"自定义"** 权限规则
3. 复制以下规则：

```json
{
  "read": true,
  "write": true
}
```

**说明**：
- `read: true` - 所有用户可读（方便分享）
- `write: true` - 所有用户可写（支持多人协作编辑）

**注意**：所有家族成员都可以编辑家谱，后保存的会覆盖先保存的。

## 步骤5：编译运行

1. 保存所有文件
2. 点击 **"编译"** 按钮
3. 测试分享功能

## 测试分享功能

### 分享者
1. 在小程序中创建一个家谱
2. 点击 **"📤 分享"** 按钮
3. 系统自动上传到云数据库
4. 点击右上角 **"..."** → **"转发给朋友"**
5. 选择一个好友或群聊

### 接收者
1. 点击群聊中的小程序卡片
2. 自动加载家谱数据
3. 进入**协作编辑模式**
4. 可以查看、添加、修改家族成员
5. 所有修改会实时同步（每10秒自动刷新）

### 协作编辑特点
- ✅ 所有人都可以编辑
- ✅ 实时同步（每10秒）
- ✅ 冲突处理：后保存的覆盖先保存的
- ✅ 每个群聊的家谱独立（基于 sharedId）
- ✅ 不需要登录授权

## 常见问题

### Q: 提示"云服务未配置"怎么办？

**A:** 检查以下几点：
1. 是否已开通云开发（步骤1）
2. `app.js` 中的环境ID是否正确（步骤2）
3. 是否创建了 `genealogies` 集合（步骤3）

### Q: 分享后接收者无法打开？

**A:** 可能原因：
1. 数据库权限设置不正确，检查步骤4
2. 网络连接问题
3. 小程序版本问题，重新编译

### Q: 如何查看我分享的家谱？

**A:** 分享的家谱会自动保存到接收者的本地，下次打开小程序可以直接查看。

### Q: 分享的家谱可以被编辑吗？

**A:** 是的，所有家族成员都可以编辑家谱。进入协作模式后，所有人都可以添加、修改、删除家族成员，所有修改会实时同步到云端。

### Q: 多人同时编辑会发生冲突吗？

**A:** 会有简单的冲突处理。如果两个人同时编辑同一个人或添加同一个人，后保存的数据会覆盖先保存的。建议避免同时编辑同一部分。

### Q: 在不同群聊中会互相影响吗？

**A:** 不会。每个群聊中的家谱卡片对应独立的 `sharedId`，数据完全隔离。群聊A编辑家谱A，群聊B编辑家谱B，互不干扰。
